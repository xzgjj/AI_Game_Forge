# AGENT_EXECUTION_PROTOCOL.md

最后更新：2026-03-24

## 目的
确保所有 AI/代理在该仓库内的修改符合“Unity + AI 生成游戏”的 5 页工作台 + 8 阶段内部流水线要求，并保证文档与实现一致。



## 必须遵守的工作流程
1. 先阅读 `CLAUDE.md`、`notes.txt`、`implementation_plan.md`。
2. 修改前先确认目标页面（页面1-5）和目标内部阶段，避免把页面和流水线混为一谈。
3. 任何结构性改动必须同步更新 `notes.txt` 与 `diff.md`，必要时同步更新 `README.md`。
4. 不得声称已实现未实现的功能；文档必须与代码一致。
5. 不得再次把产品退回成线性 8 步表单向导。
6. 具体页面职责、状态变化、可见反馈和扩张规则以 `implementation_plan.md` 为准。



## 输出质量要求
- 所有新功能必须绑定到具体页面、状态对象、快照或版本记录。
- 页面新增内容时必须说明用户行为路径、系统状态变化和可见反馈。
- Unity 相关功能必须包含：模板、UPM 注入、导入校验、回滚路径。
- AI 生成相关功能必须支持：玩家控制、敌人 AI、碰撞、状态机，以及内容画布上的角色、台词和场景节点。



## 文件与数据规范
- 结构化数据必须保存为 JSON 并入库。
- 项目、工作区、内容节点、资源、版本快照和发布记录要分层建模。
- 生成资产必须有 manifest 与 GUID 追踪。
- 审计记录必须包含：时间、页面或阶段、输入、输出、模型与成本估算。



## 测试要求
- 新增功能需提供最小验证路径。
- 若无法运行测试，必须在报告中说明原因。



## 禁止事项
- 未经讨论改变 5 页结构或内部阶段定义。
- 将 AI 生成结果直接写入代码而不保留原始输入/输出。
- 引入不透明的自动化脚本或未记录的依赖。
- 用新的表单页替代页面级工作台和版本快照机制。
