# AGENT_EXECUTION_PROTOCOL.md

最后更新：2026-03-13

## 目的
确保所有 AI/代理在该仓库内的修改符合“Unity + AI 生成游戏”的 8 步流程与自动对接要求，并保证文档与实现一致。

## 必须遵守的工作流程
1. 先阅读 `CLAUDE.md`、`notes.txt`、`implementation_plan.md`。
2. 修改前确认目标步骤（Step1-8），避免跨步修改。
3. 任何结构性改动必须同步更新 `notes.txt` 与 `diff.md`。
4. 不得声称已实现未实现的功能；文档必须与代码一致。

## 输出质量要求
- 所有新功能必须绑定到具体步骤与产物。
- Unity 相关功能必须包含：模板、UPM 注入、导入校验、回滚路径。
- AI 生成相关功能必须支持：玩家控制、敌人 AI、碰撞、状态机。

## 文件与数据规范
- 结构化数据必须保存为 JSON 并入库。
- 生成资产必须有 manifest 与 GUID 追踪。
- 审计记录必须包含：时间、步骤、输入、输出、模型与成本估算。

## 测试要求
- 新增功能需提供最小验证路径。
- 若无法运行测试，必须在报告中说明原因。

## 禁止事项
- 未经讨论改变 8 步流程或页面结构。
- 将 AI 生成结果直接写入代码而不保留原始输入/输出。
- 引入不透明的自动化脚本或未记录的依赖。
