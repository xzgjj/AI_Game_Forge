# implementation_plan.md

最后更新：2026-03-13

## 目标
把“Unity + AI 生成游戏”的 8 步流程落地为可用产品，确保每一步有清晰产物、可回滚、可审计。

## 阶段计划

### 阶段 1：流程引擎与 8 步 UI（MVP-1）
- 建立 `WizardState` 状态机与步骤校验
- 8 个独立页面 + 顶部进度条 + 底部内容区
- 步骤产物以结构化 JSON 存储（SQLite）
- 最小审计记录（Step 输入/输出、AI 调用）

验收：完整 8 步可走通，Step1-3 产物可持久化。

### 阶段 2：Unity Bridge（MVP-2）
- 生成 Unity 项目模板（Scene + 目录结构）
- 脚本以 UPM 包注入，带 manifest/GUID 映射
- Unity Editor 一键导入与编译校验
- 输出目录监听与自动刷新机制

验收：点击“初始化项目”后，Unity 可打开并运行空场景，脚本可编译。

### 阶段 3：AI 生成核心脚本（MVP-3）
- 玩家控制脚本生成
- 敌人 AI + 状态机生成
- 碰撞检测与事件系统生成
- 自动编译检查与错误回传

验收：脚本可编译并在 Unity 场景中运行。

### 阶段 4：人物与资源生成（MVP-4）
- 角色画像 + 台词生成与绑定
- 角色 prefab/动画占位生成
- 场景/音效/UI 占位资源生成

验收：角色与资源可被 Unity 场景引用。

### 阶段 5：玩法迭代与发布（MVP-5）
- 玩法迭代记录与版本回滚
- 数值调优与技能系统迭代
- 一键打包发布与版本摘要

验收：至少一次完整迭代 + 可回滚 + 可发布 Demo。

## 风险控制
- 生成脚本编译失败必须在 Step4 阶段拦截。
- 资源命名与 GUID 映射必须稳定。
- 文档与实现必须同步，禁止“报告先行、实现缺失”。
