# Web UX Review

最后更新：2026-04-02

## 目的
用于核对当前桌面端工作流的页面结构、用户行为、反馈态和回退路径，作为 Figma 与代码协作的审阅基线。

## 当前界面范围
1. 登录页
2. 仪表盘
3. 配置向导
4. AI 画布

## 页面核对点
### 登录页
- 是否提供邮箱登录、微信登录、手机登录、邮箱注册和第三方 OAuth 入口
- 是否展示错误态与加载态
- 是否允许回到同一页重新尝试

### 仪表盘
- 是否提供项目管理、配置向导和 AI 画布的入口
- 是否保留总览、设置和 API 统计信息
- 项目“打开”是否能进入后续工作流

### 配置向导
- 是否有结构化步骤和清晰的进度反馈
- 是否展示 Unity 初始化和校验状态
- 是否保留表单与结构化预览的并置关系

### AI 画布
- 是否有输入、预览和版本历史
- 是否有返回仪表盘的入口
- 是否能在生成后看到状态变化

## 当前风险
- 代码流已打通，但仍未实现完整 5 页工作台。
- 设计稿不应把 8 个内部阶段误画成 8 个用户页面。
- 如果要从当前流扩展到完整工作台，先更新本文件，再更新代码与设计。

## 使用方式
1. 先用本文件核对页面行为。
2. 再用 `doc/figma_ui_spec.md` 核对视觉与布局。
3. 每次结构性变更都同步更新 `notes.txt` 和 `diff.md`。
