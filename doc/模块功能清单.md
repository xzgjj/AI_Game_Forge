# GameCraft AI Studio 模块与功能清单



基于《GameCraft AI Studio Architecture Report》的详细架构设计，本文档系统性地拆解了项目的所有模块和功能需求。



## 项目概述

**项目名称**: GameCraft AI Studio
**项目类型**: 跨平台桌面应用（Windows/macOS）
**核心目标**: 通过AI辅助降低游戏开发门槛，让创意者无需编程即可快速原型化游戏创意，实现"创意即游戏"的无缝转化。



## 一、核心功能需求



### 1.1 多元登录系统
- **第三方OAuth登录**: GitHub、Google
- **新增登录方式**:
  - 微信扫码登录（微信开放平台OAuth 2.0）
  - 手机号验证登录（短信服务商集成）
  - 邮箱注册登录（自有验证系统 + SMTP）
- **本地用户配置持久化**



### 1.2 配置向导
- 分步式游戏配置流程:
  - 游戏类型选择（RPG/冒险/解谜等）
  - 美术风格选择（像素/手绘/3D卡通等）
  - 叙事风格设置（轻松/史诗/悬疑等）
  - 目标平台选择（PC/手机/主机）
  - 高级设置（可选）
  
  

### 1.3 AI协作工作区
- **实时AI生成游戏元素**:
  - 角色设计
  - 场景描述
  - 对话台词
- **可视化编辑与反馈循环**: 修改→重新生成→再修改
- **动作效果预览与参数调整**



### 1.4 项目管理
- 多版本管理
- 导出配置（支持主流游戏引擎格式）



### 1.5 审计追踪
- 完整记录所有AI调用
- 用户操作历史记录
- 生成内容版本历史



### 1.6 AI API管理与统计面板
- **多厂商API接入**: OpenAI、Claude、智谱AI、百度文心、本地Ollama等
- **Token消耗实时统计与报表**
- **API成本分析与预算管理**
- **智能路由与故障转移**

## 三、新增模块详细设计



### 3.1 认证服务模块详细设计

#### 认证提供商标识
```rust
pub enum AuthProvider {
    OAuth2(OAuthConfig),      // 微信、GitHub、Google
    Phone(PhoneAuthConfig),   // 短信验证
    Email(EmailAuthConfig),   // 邮箱验证
    Local(LocalAuthConfig),   // 本地账号
}
```



#### 核心功能接口

1. **微信扫码登录**: `wechat_login(auth_code: String) -> Result<AuthResult>`
2. **手机验证登录**: `phone_login(phone: String, code: String) -> Result<AuthResult>`
3. **邮箱注册登录**: `email_register(email: String, password: String) -> Result<AuthResult>`
4. **第三方OAuth登录**: `oauth_login(provider: String, code: String) -> Result<AuthResult>`
5. **会话管理**: `logout(session_id: Uuid)`, `validate_session(token: String)`
6. **安全控制**: 登录尝试限制、会话超时管理、令牌刷新



### 3.2 API管理服务详细设计



#### 核心功能接口
1. **提供商注册**: `register_provider(name: String, provider: Box<dyn AIProvider>)`
2. **智能路由**: `smart_route(request: AIGenerationRequest) -> Result<AIGenerationResponse>`
   - 根据任务类型、成本、性能、可用性选择最佳提供商
   - 支持故障自动转移
   - 成本优化策略

3. **统计分析**: `get_usage_stats(period: StatsPeriod) -> Result<UsageStats>`
   - 按时间段统计（日/周/月/自定义）
   - 按提供商统计
   - 按项目统计
   - 成本趋势分析

4. **预算管理**: `set_budget_limit(provider: String, monthly_limit: f64) -> Result<bool>`
   - 多级预算设置（用户级、项目级、提供商级）
   - 预算告警系统
   - 超额限制策略

5. **监控告警**: `check_budget_alerts() -> Vec<BudgetAlert>`
   - 阈值告警（50%、80%、95%）
   - 异常使用检测
   - 优化建议生成

## 五、接口定义



### 5.1 认证接口 (auth.rs)
```rust
// 登录方法枚举
pub enum LoginMethod {
    WechatQr,      // 微信扫码
    PhoneCode,     // 手机验证码
    EmailPassword, // 邮箱密码
    OAuth(String), // OAuth提供商
}

// 登录请求
pub struct LoginRequest {
    pub method: LoginMethod,
    pub credentials: HashMap<String, String>,
}

// 认证响应
pub struct AuthResponse {
    pub success: bool,
    pub user: Option<UserInfo>,
    pub token: Option<String>,
    pub error: Option<String>,
}

// IPC命令接口
#[tauri::command]
pub async fn login(app_handle: AppHandle, request: LoginRequest) -> Result<AuthResponse, String>;

#[tauri::command]
pub async fn logout(app_handle: AppHandle, session_id: Uuid) -> Result<bool, String>;

#[tauri::command]
pub async fn register_email(app_handle: AppHandle, email: String, password: String, verification_code: String) -> Result<AuthResponse, String>;
```

### 5.2 API统计接口 (api_stats.rs)
```rust
// 使用统计数据结构
pub struct UsageStats {
    pub period: StatsPeriod, // 日/周/月/自定义
    pub total_requests: i64,
    pub total_tokens: i64,
    pub total_cost: f64,
    pub by_provider: Vec<ProviderStats>,
    pub by_project: Vec<ProjectStats>,
    pub cost_trend: Vec<DailyCost>,
}

// 预算告警
pub struct BudgetAlert {
    pub level: AlertLevel, // 信息/警告/危险
    pub message: String,
    pub threshold: f64,
    pub current: f64,
    pub suggested_action: String,
}

// IPC命令接口
#[tauri::command]
pub async fn get_usage_stats(app_handle: AppHandle, period: StatsPeriod) -> Result<UsageStats, String>;

#[tauri::command]
pub async fn set_budget_limit(app_handle: AppHandle, provider: String, monthly_limit: f64) -> Result<bool, String>;

#[tauri::command]
pub async fn get_budget_alerts(app_handle: AppHandle) -> Result<Vec<BudgetAlert>, String>;
```

## 总结



本清单全面梳理了GameCraft AI Studio项目的所有模块和功能需求，共涵盖**10个大类、超过50个核心模块**。项目采用分层架构设计，具有以下特点：

1. **模块化程度高**: 各层模块职责清晰，便于独立开发和测试
2. **扩展性强**: 新增的认证和API管理模块采用插件化设计
3. **安全性优先**: 多层安全机制，符合企业级应用标准
4. **成本可控**: 完善的预算管理和成本优化系统
5. **用户体验优秀**: 流畅的交互设计和iPhone级动画效果

项目技术栈选择前瞻且合理，为"创意即游戏"的核心目标提供了坚实的技术基础。
