## 2026-04-02 处理 ESLint 警告与 flat config 缺失
1. **问题定位**：`npm run lint` 失败的根因是 ESLint 9 缺少 `eslint.config.js`，不是业务代码本身报错。
2. **处理原则**：先恢复 lint 基础设施，再看代码级警告，避免在配置缺失时误判大量噪音。
3. **落地结果**：新增 `eslint.config.js`，接入 flat config、Svelte、TypeScript 和浏览器/Node 全局变量配置。
4. **验证结果**：`npm run lint` 与 `npm run check` 均已恢复正常。

## 2026-04-02 处理 330 个 Svelte 警告
1. **问题定位**：大量警告来自 Svelte 5 迁移后的旧语法混用，包括 `on:` 事件、`createEventDispatcher`、`export let`、未使用 `$state` 的本地状态，以及 `ConfigWizard` 里的 `legacy reactive statement`。
2. **处理原则**：按 Svelte 5 官方迁移方式统一处理，优先迁移事件和响应式状态，而不是压制警告。
3. **落地结果**：将组件事件改为 callback props、DOM 事件改为 `onclick/onsubmit`、本地可变状态改为 `$state`，并将 `ConfigWizard` 的 `$:` 改为 `$derived`。
4. **验证结果**：`npm run check`、`npm run lint`、`npm run build` 已全部通过，Svelte 警告清零。

## 当前状态审查（新规划对齐）
1. **用户页面已重新定义**：前台不是线性 8 步向导，而是 5 个用户页面，分别承载新建项目、资源导入、内容画布、协作优化和发布版本记录。
2. **内部阶段与页面必须分离**：8 个阶段保留为内部生成与校验流水线，不再等同于 UI 页面数量。
3. **当前实现仍偏旧模型**：现有 `ConfigWizard` 和 `WizardState` 仍是线性表单与扁平快照，暂时不够支撑画布节点、版本分支和对象级 diff。
4. **Unity 对接仍需补强**：目前可用的是初始化、UPM 注入、基础校验和可选 BatchMode 验证，还缺少资源同步、最小场景验证、回滚与发布审计。
5. **新增内容必须版本化**：角色、台词、资源和场景都应作为可追踪对象，新增内容不能直接覆盖旧版本。

## 已确认的核心目标
- 固定 5 个用户页面，页面职责清晰，状态变化和用户反馈可见。
- 8 个内部阶段用于实现与校验，不与页面数量混淆。
- 自动对接 Unity（项目模板 + 脚本包 + 资源同步 + 校验 + 回滚）。
- AI 生成能力必须覆盖：玩家控制、敌人 AI、碰撞、状态机、核心循环、数值、技能，并且能落到内容画布上的对象节点。
